# Real-data sources and methodology

The checked-in catalog contains 216 real, publicly documented Virginia ecosystem assets verified on July 17, 2026. It is generated by `scripts/build_real_asset_catalog.py` and stored in `data/virginia_real_assets.json` so changes can be reviewed before they reach the database.

## Coverage

| Source group | Records | Location treatment |
| --- | ---: | --- |
| FAA and Virginia Department of Aviation public-use airports | 64 | Published airport reference coordinates |
| Virginia Military Factbook installations | 30 | Generalized locality coordinates |
| Curated research, workforce, company, program, and infrastructure assets | 122 | Generalized public campus or locality coordinates |

The catalog currently covers 12 Virginia ecosystem regions and five record types. Every record has an unmanned-systems relevance statement, at least one public evidence URL, a verification date, and normalized taxonomy values.

## Inclusion rules

- The named organization, facility, program, or infrastructure asset must be real and publicly documented.
- The record must have direct unmanned-systems relevance or a clear enabling role defined by the project specification, such as aviation infrastructure, defense customer access, workforce, ports, advanced manufacturing, or test logistics.
- Public descriptions must not include classified, controlled, live-operational, vulnerability, or security-procedure information.
- Military and federal records identify only publicly documented installations and use generalized points.
- Airport inclusion does not imply permission to fly UAS there. Operators remain responsible for all airport, airspace, and regulatory authorization.

## Principal sources

- Federal Aviation Administration Airports Feature Service
- Virginia Department of Aviation Public-use Airport Directory
- Virginia Secretary of Veterans and Defense Affairs Military Factbook
- Virginia Economic Development Partnership unmanned-systems industry profile
- Virginia Innovation Partnership Corporation Unmanned Systems Center
- Official university, NASA, Port of Virginia, and research-laboratory pages

One authoritative overview may support multiple related records, such as a university, its laboratory, and a documented program. Public profile pages expose the source used for each individual record.

## Update workflow

```bash
python scripts/build_real_asset_catalog.py
git diff -- data/virginia_real_assets.json
python manage.py seed_real_data --prune
python manage.py test tests.backend.test_real_catalog
```

Review generated changes before seeding. The `--prune` option removes only catalog-managed records absent from the new snapshot; it does not remove manually curated records without the catalog provenance marker.
